# store/

Persistence layer. All SQL is generated by sqlc from `queries/*.sql` into `sqlite/`; hand-written Go code in this package is the thin domain-facing wrapper (param/result struct mapping, JSON capability marshaling).

## Layout

- `queries/*.sql` — sqlc input, one file per entity.
- `sqlite/` — sqlc-generated Go. **Committed to git.** Never edit by hand; never import from outside this package.
- `migrations/*.sql` — golang-migrate up/down pairs. Schema changes go here.
- `{entity}.go` — per-entity wrapper methods on `*DB` that translate sqlc rows to domain types.
- `store.go` — domain-facing param/result types (`CreateAutomationParams`, `Scene`, `UserRef`, ...).
- `mapper.go` — shared mapping helpers (`userRefFromPtrs`, `marshalCapabilities`, `unmarshalCapabilities`, `MarshalLightCommand`, `UnmarshalLightCommand`).
- `db.go` — `*DB` struct wrapping `*sqlite.Queries` and its constructor.

## sqlc workflow

- `make sqlc` — regenerate `sqlite/` from `queries/*.sql` + `migrations/`. Uses pinned sqlc v1.31.0 (see `sqlc.yaml`).
- `make sqlc-check` — regenerates and fails if the committed output drifted. Wired into `prepare-for-commit` and CI.

## Query gate patterns

sqlc's SQLite engine has two relevant limitations that shape how queries are written:

1. **No data-modifying CTEs.** `WITH x AS (UPDATE … RETURNING *) SELECT …` does not parse. Post-insert/update reads that need a joined row (e.g. creator attribution) use a separate `Get*` query.
2. **CASE WHEN type inference is unreliable.** Param types collapse to garbage when `sqlc.arg` appears inside `CASE WHEN … THEN …` branches. Use the COALESCE pattern below instead.

The gate idioms — copy these when adding a new query:

| Situation | Pattern |
|---|---|
| Partial UPDATE (skip unchanged fields) | `col = COALESCE(sqlc.narg('x'), col)` → Go param `*T`. Nil leaves the column alone. |
| Clearing a nullable column to NULL | Dedicated `:exec` query (e.g. `ClearAutomationIcon`). COALESCE can't distinguish "leave alone" from "set to NULL". |
| Optional equality filter on a **nullable** column | `CAST(sqlc.narg('x') AS TEXT) IS NULL OR col = CAST(sqlc.narg('x') AS TEXT)`. The CAST pins the Go param type to `*string`; without it sqlc emits `interface{}`. |
| Optional comparison filter on a **NOT NULL** column | `col ≥ COALESCE(sqlc.narg('x'), col)` — self-neutral when narg is NULL. |
| Dynamic `IN (…)` with "match all" fallback | `json_array_length(CAST(sqlc.arg('x') AS TEXT)) = 0 OR col IN (SELECT value FROM json_each(CAST(sqlc.arg('x') AS TEXT)))`. Caller passes a JSON-array string; `[]` means "match all". |
| Dynamic `NOT IN (…)` | `col NOT IN (SELECT value FROM json_each(CAST(sqlc.arg('x') AS TEXT)))`. Caller passes `[]` to exclude nothing (NOT IN over an empty subquery is TRUE for every row). |
| Optional LIMIT | `LIMIT IIF(CAST(sqlc.arg('lim') AS INTEGER) > 0, CAST(sqlc.arg('lim') AS INTEGER), -1)` — SQLite treats LIMIT -1 as unbounded. |
| JSON / text param type hint | Wrap the arg in `CAST(sqlc.arg(…) AS TEXT)` to force `string` in Go. |

**Do not mix named args (`sqlc.arg`/`sqlc.narg`) with bare `?` in the same query** — sqlc 1.31.0 produces off-by-one parameter indices when it does. If a query uses any named arg, name all of them.

## Domain/DB type separation

Consumers of this package depend on a narrow interface they define locally, listing only the methods they call; `*DB` satisfies them structurally. There is no global `Store` interface — if you find yourself wanting to type something as `store.Store`, declare an interface in your own package instead.

Type overrides in `sqlc.yaml` map TEXT columns with domain meaning (`devices.id`, `devices.source`, `devices.type`, `scene_actions.target_type`, `group_members.member_type`) to their `internal/device` types directly, so mapping is usually a struct copy.
